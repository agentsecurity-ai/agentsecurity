Adopt AgentSecurity

Get your agent secured in 60 seconds. Three steps, no security expertise required.

Quick Start

1 Install the CLI

pip install agentsec

2 Initialize Your Policy

# Interactive mode — prompts for tier and name
agentsec init

# Or specify directly
agentsec init --tier standard --name my-agent

This creates AGENTSECURITY.md in your project root with sensible defaults. The interactive selector shows all four tiers with descriptions.

3 Customize and Validate

# Edit AGENTSECURITY.md to match your agent's actual tools

# Validate the policy
agentsec validate .

# Scan your codebase for violations
agentsec check .

# Get framework-specific recommendations
agentsec suggest .

4 Lock and Protect

# Generate tamper-protection lock file
agentsec lock .

# Verify integrity (use in CI/CD)
agentsec verify .

The lock file stores a SHA-256 hash of your policy. If an agent or process modifies AGENTSECURITY.md, agentsec verify will catch it and exit with code 1.

CI/CD Integration

GitHub Action

name: AgentSecurity Check
on: [pull_request]
jobs:
  agentsec:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install agentsec
        run: pip install agentsec
      - name: Verify policy integrity
        run: agentsec verify . --lock .agentsecurity.lock
      - name: Validate policy
        run: agentsec validate .
      - name: Check codebase
        run: agentsec check . --fail-on high
      - name: Generate report
        run: agentsec report . --format json -o agentsec-report.json
      - uses: actions/upload-artifact@v4
        with:
          name: agentsec-report
          path: agentsec-report.json

Pre-commit Hook

# .pre-commit-config.yaml
repos:
  - repo: local
    hooks:
      - id: agentsec-validate
        name: Validate AGENTSECURITY.md
        entry: agentsec validate .
        language: system
        pass_filenames: false

Framework Integration

Agent System Prompt Injection

Use agentsec to-prompt to generate an XML snippet for your agent's system prompt:

agentsec to-prompt .

Output (~50-100 tokens):

<agent_security_policy>
  <name>my-agent</name>
  <tier>standard</tier>
  <enforcement>warn</enforcement>
  <description>Processes support tickets</description>
  <location>./AGENTSECURITY.md</location>
</agent_security_policy>

LangChain

Add the security policy as system context when initializing your agent:

from agentsec.parser import parse_policy

policy = parse_policy(".")
system_context = f"""
You are operating under security tier: {policy.security_tier}
Enforcement mode: {policy.enforcement}
Declared tools: {', '.join(policy.declared_tool_names)}
"""

Claude Code

Place AGENTSECURITY.md in your project root alongside CLAUDE.md. Claude Code will discover it automatically when configured.

Framework Auto-Detection

Run agentsec suggest to auto-detect your agent framework and get tailored recommendations:

agentsec suggest .

Detected frameworks include:

  • Python: LangChain, CrewAI, AutoGen, OpenAI SDK, Anthropic SDK, Gemini SDK, LlamaIndex, LiteLLM, Haystack, PydanticAI, and more
  • JavaScript/TypeScript: LangChain.js, Vercel AI SDK, OpenAI SDK, Anthropic SDK, Mastra, LlamaIndex.TS
  • Agent Platforms: Claude Code, Cursor, GitHub Copilot

The suggest command also generates enforcement guides specific to each framework, showing exactly how to make your agent respect its security boundaries.

Tamper Protection

Agents must never modify their own security policy. Use the lock/verify workflow to enforce this:

# Generate a lock file with SHA-256 hash
agentsec lock .

# Verify the policy hasn't been tampered with
agentsec verify . --lock .agentsecurity.lock

Add .agentsecurity.lock to version control and run agentsec verify in your CI pipeline to block deployments if the policy has been altered.

Scan Exceptions

Use .agentsecignore to exclude files from scanning (vendor code, generated code, etc.):

# .agentsecignore
vendor/*
*.generated.ts
prisma/client/
neutralino/

Patterns follow the same syntax as .gitignore. Lines starting with # are comments. Test files (*.test.ts, __tests__/) are automatically scanned with reduced severity for secrets.

CLI Reference

# Initialize from template (interactive or with flags)
agentsec init [--tier basic|standard|strict|regulated] [--name NAME]

# Validate AGENTSECURITY.md against the spec
agentsec validate [path]

# Scan codebase for violations
agentsec check [path] --fail-on high|medium|low

# Detect frameworks and get recommendations
agentsec suggest [path] --format text|json

# Generate tamper-protection lock file
agentsec lock [path] --output .agentsecurity.lock

# Verify policy integrity against lock file
agentsec verify [path] --lock .agentsecurity.lock

# Generate security scorecard
agentsec report [path] --format text|json|badge

# Output policy properties as JSON
agentsec read-properties [path]

# Generate system prompt XML
agentsec to-prompt [path]

Replace standard with your tier. Color options:

  • basic — blue
  • standard — green
  • strict — yellow
  • regulated — red

Templates

Pre-built templates are available in the repository:

  • Basic — Minimal, ~30 lines
  • Standard — Balanced, ~80 lines
  • Strict — Maximum enforcement, ~120 lines
  • Regulated — Full compliance, ~150 lines

Examples

  • LangChain Research Agent — Standard tier, read-only web search
  • Claude Code Project — Standard tier, file + shell access
  • Financial Processor — Strict tier, Stripe API + PCI scope
