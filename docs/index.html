Guardrails for AI Agents

The open-source standard for defining unbreakable security boundaries for AI agents. Add an AGENTSECURITY.md to your project to instantly enforce safe defaults, tool allowlisting, and runtime sandboxing before your autonomous agent even boots.

Quickstart

# Install from source
python3 -m pip install "git+https://github.com/agentsecurity-ai/agentsecurity.git@main#subdirectory=packages/agentsec"

# Initialize security policy
agentsec init

# Scan codebase for policy violations
agentsec check .

# Generate token-efficient prompt snippet
agentsec to-prompt --verbose --compact
# Install (PowerShell)
python -m pip install "git+https://github.com/agentsecurity-ai/agentsecurity.git@main#subdirectory=packages/agentsec"

# Initialize
agentsec init

# Scan
agentsec check .

# Prompt snippet
agentsec to-prompt --verbose --compact
# Run on-demand without install
docker run -it --rm -v "$(pwd):/app" python:3.11-slim /bin/bash -c \
 "pip install agentsec && agentsec scan /app"

How It Works in Practice

1. Define the Contract (The Rulebook)

You configure an AGENTSECURITY.md file declaring exactly what your AI agent is allowed to do. For example, explicitly permitting internal docs reading while strictly forbidding shell commands.

constraints:
  hard_no:
    - "Never execute arbitrary shell commands"
tools:
  - name: internal_docs_reader
    permission: read_only
    scope:
      allowed_paths: ["/docs/*"]

2. CI/CD Enforcement (The Shield)

If a teammate opens a Pull Request that deletes a security constraint or adds an overly broad wildcard permission (*), the GitHub Action automatically fails the pull request and blocks the insecure code from merging. 

FAIL [HIGH] Tool 'file_system' has wildcard scope ('*'). Narrow the scope to specific resources.
RESULT: FAIL (high severity findings detected)

3. Runtime Protection (The Engine)

Before your agent executes a tool at runtime, your python app strictly evaluates the AGENTSECURITY.md frontmatter block. If your LLM gets prompt-injected or goes rogue, the physical execution wrapper intercepts and denies the function call.

if tool.name not in policy['tools']:
    raise SecurityPolicyViolation("Agent specifically attempted to use an unauthorized tool.")

Why AgentSecurity?

Secure by Design

Define security boundaries before your agent runs, not after a breach. Architecture-level safety, not runtime patches.

Framework Agnostic

Works with LangChain, CrewAI, AutoGen, Claude Code, and any future framework. One standard, everywhere.

Progressive Disclosure

Metadata loads at startup (~100 tokens). Full policy on activation. Detailed references on demand. Minimal context overhead.

Honest About Limits

CI/CD Ready
